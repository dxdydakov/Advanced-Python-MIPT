Что такое git и чем он отличается от GitHub?
Git — распределённая система контроля версий, которая даёт возможность разработчикам отслеживать изменения в файлах и 
работать над одним проектом совместно с другими людьми.
GitHub — сервис онлайн-хостинга репозиториев, обладающий всеми функциями распределённого контроля версий и 
функциональностью управления исходным кодом — всё, что поддерживает Git.

Зачем использовать git и подобные ему системы в больших проектах?
Скорость, простой дизайн, поддержка нелинейной разработки 

Какие основные термины используются при работе с git?
Ветки, репозитории, commit, push, pull, rebase, merge

В каких случаях имеет смысл создавать различные ветки проекта?
Исправить баги или внести новые фичи

В чем отличия merge и rebase? В чем преимущества и недостатки последнего из них?
merge: существующие ветки никак не изменяются 
rebase: перебазирование функциональной ветки feature на главную ветку main ( ветка feature оказывается поверх главной ветки main)
Преимущества: более чистая история проекта (линейная), устраняет ненужные коммиты слияния, необходимые для git merge.
Недостатки: жертвуем безопастностью и отслеживаемостью. При выполненнии rebase теряется контекст, доступный в коммите со слиянием:
не можем видеть, когда вышестоящие изменения были включены в функционал

В чем отличия централизованных СКВ от децентрализованных?
Централизованные  системы контроля версий предполагают сохранение версий проектов на  общий сервер, 
В децентрализованных системах контроля версий при каждом копировании удалённого репозитория (расположенного на сервере) 
происходит полное копирование данных в локальный репозиторий (установленный на рабочем компьютере). 
Каждая копия содержит все данные, хранящиеся в удалённом репозитории. 
В случае, возникновения технической неисправности на стороне сервера, удаленный репозиторий можно перезаписать 
с любой сохраненной копии.с которого потом получают нужные версии клиенты.
